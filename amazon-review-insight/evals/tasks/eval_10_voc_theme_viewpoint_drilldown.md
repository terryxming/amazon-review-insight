# Eval 10: VOC Theme Viewpoint Drilldown

输入：单个 Amazon US ASIN 的 Sorftime Review 样本。

检查：

1. 每个前台 `voc_theme` 都包含 `viewpoints[]`。
2. 每个 viewpoint 都有 `viewpoint_id`、`viewpoint_name`、`review_count`、`sample_size`、`percentage`、`role`、`viewpoint_polarity`、`reason`、`business_meaning` 和 `evidence`。
3. viewpoint `percentage` 使用 Review 样本数作为分母。
4. VOC 主题地图展示每个主题下的观点分布、提及数量和占比。
5. 点击 VOC 主题卡片后在新标签页打开同一 HTML 内的主题详情页，当前主报告页不得滚动跳转。
6. 主题详情页顶部 sticky 面板在同一个卡片内展示所属 VOC 主题和观点筛选入口。
7. sticky 面板的主题区展示核心问题、归因假设、业务含义和运营动作。
8. 新标签页只展示当前主题详情，不展示目录、主报告其它章节或其它主题详情。
9. sticky 面板下方不再另起卡片重复展示同一主题摘要。
10. 主题详情页默认展示该主题相关的全量 Review 原文和完整中文翻译。
11. 主题详情页提供观点筛选入口；点击某个观点后，在当前标签页只展示该观点相关的全量 Review。
12. 原文和译文中的完整证据句使用语义颜色高亮，且不保留关键词高亮字段。
13. 同一主题下观点允许多标签重叠，报告中说明占比允许合计超过 100%。
14. Excel 包含 `VOC主题观点` 和 `VOC观点评论明细`，可用于观点级人工复核。
15. 左侧导航必须在 `4. VOC 主题地图` 下展示带序号的 `正向主题`、`负向主题`、`未满足的机会点` 三个二级导航；该一级导航默认展开并支持折叠；正文中不应额外渲染“主题与观点分布”副标题。
16. HTML 中的 VOC 观点分布表不得展示 `role`、角色列或角色徽标；角色只保留在分析 JSON 和 Excel 中用于复核。
17. 主题块、详情 sticky 面板和观点筛选入口必须共享同一套字段口径，迭代时不得出现旧版重复主题卡、旧角色列或旧多列布局残留。
18. 主报告一级章节标题下方不展示说明文字；VOC 主题地图标题右上角必须有问号 tooltip。
19. HTML 不再包含移动端媒体查询或移动端特化样式。
20. VOC 主题地图和详情 sticky 面板必须展示“运营优先级”及动作语义，不得裸展示 `优先级：P0/P1/P2`。
21. VOC 主题地图 tooltip 必须解释 P0/P1/P2 是运营动作顺序而不是严重度：正向主题用于放大转化，负向或风险主题用于止损，P2 通常用于排期优化或低成本补齐。
22. VOC 主题地图必须按 `正向主题`、`负向主题`、`未满足的机会点` 三组穷举全部前台主题，三组主题数量之和等于 `voc_themes.length`，不得遗漏或重复主题。
23. 关键结论卡片必须像 VOC 主题卡片一样整体可点击，并在新标签页打开 `#insight-detail-{dimension}` 详情页。
24. 关键结论详情页必须进入独立路由模式，只展示当前关键结论详情，并提供类型筛选入口。
25. 点击关键结论详情页中的类型后，必须在当前标签页筛选出该类型相关的全量 Review 原文和完整中文翻译。
26. 主报告关键结论和 VOC 主题卡片中的证据区必须优先展示 3 条不同的完整代表性 Review 原文和完整中文译文；不得把关键词、短句或 `evidence[]` 片段当作代表性原文。
